Git, Github und Atom

Dies ist ein winzig kleines Beispiel zur Quellcodeverwaltung mit Git und Github zusammen mit dem Atom Editor.

Git-Repository initialisieren

Zunächst brauchen wir ein neues Atom-Projekt mit ein paar Quellcodedateien darin. Wie du weißt, ist ein Projekt in Atom einfach nur ein Ordner auf einer Festplatte. Lege diesen nun an und erzeuge eine Datei, zum Beispiel mit dem Namen index.html. Danache geht es wie folgt wieter:

  1. Öffne den neuen Ordner in Atom mit File → Add Project Folder
  2. Öffne den Tab mit View → Toggle Git Tab
  3. Initialisiere das Repository, indem du im Git-Tab auf Initialize Repository klickst.

Dein erster Commit

Innerhalb des Git-Tab tauchen die neuen Dateien in einer Liste mit dem Namen Unstaged Changes auf. Damit ist gemeint, dassd diese Änderungen bisher noch nicht im Repository vorhanden sind und ihm auch nicht hinzugefügt werden. Um dies zu tun, gehe wie folgt vor:

  1. Klicke oben rechts auf Stage All, um alle Änderungen zu übernehmen. Alternativ kannst du auch nur einzelne Dateien oder sogar nur die markierten Zeilen innerhalb einer Datei übernehmen. Dadurch werden die Änderungen zum Einchecken vorgemerkt.
  2. Gib unten bei Commit Message eine ausführliche Beschreibung deiner Änderungen ein. Mit dem Symbol unten rechts im Textfeld kannst du es dabei vergrößern, um mehr Platz zu bekommen.
  3. Klicke auf Commit, um die vorgemerkten Änderungen einzuchecken.

Auf diese Weise kannst du nun nach Belieben Änderungen an deinem Projekt vornehmen und diese in das git-Repository einchecken. Beachte dabei aber, dass ein „Commit” immer eine abgeschlossene Änderung umfassen sollte. Checke also nicht gleich jede kleinste Änderung mit einem eigenen „Commit” ein. Mach die Commits aber auch nicht zugroß.

Die erste Zeile der Commit-Message ist ihre Überschrift. Sie sollte kurz und präzise den wesentlichen Inhalt der Änderung wiedergeben. Alle folgenden Zeilen sind der Inhalt der Commit-Message. Nimm dir ruhig die Zeit, deine Änderungen hier ausführlich zu beschreiben.

Hochladen auf Github

Bisher arbeiten wir nur lokal, so dass du zwar ein git-Repository auf deinem Rechner hast, außer dir aber niemand die Versionshistorie deines Projekts nachvollziehen kann. Zeit dies zu ändern, und sich in Github anzumelden.

  1. Besuche die Seite github.com und lege einen neuen Account an, falls du noch keinen hast.
  2. Abschließend klicke auf New repository, um ein neues Repository auf github anzulegen.
  3. Gib einen Namen und eine Beschreibung für dein neues Repository ein. Alle anderen Einstellungen musst du so lassen, wie sie sind. Füge an dieser Stelle dem Repository keine Dateien hinzu!
  4. Klicke nun auf Create repository.
  5. Wenn alls gut ging, siehst du nun eine Anleitung, mit welchen Befehlen du Quellcode hochladen kannst. Siehst du die Seite nicht, lösche das Repository und lege es erneut an.
  6. Klicke unter der Überschrift Quick setup auf SSH und kopiere die angezeigte git-Adresse in die Zwischenablage.

Weiter geht es im Atom Editor.

  1. Wähle im Menü den Punkt Packages → Git Plus → Run aus.

  2. Gib folgenden Befehl in das Befehlsfenster ein:

    remote add origin git@github.com/…

    Den letzten Teil musst du dabei durch die eben kopierte URL ersetzen!

  3. Öffne das Git-Befehlsfenster erneut und gib folgenden Befehl ein:

    push -u origin master
  4. Wenn du die Seite in Github neulädst, solltest du nun deinen Quellcode dort sehen.

Dein Repository ist nun mit Github verknüpft. Technisch gesehen handelt es sich dabei um ein lokales Repository auf deinem Rechner sowie ein entferntes Repository auf Github. Beide Repositories haben ihren eigenen Versionsverlauf, den du mit der nachfolgenden Anleitung synchronisieren kannst.

Weitere Dateien anlegen und hochladen

An dieser Stelle wollen wir nun an dem Projekt weiterarbeiten und zwei wichtige Dateien anlegen: README.md und LICENSE. Diese Dateien haben folgende Bedeutung:

  • README.md: Diese Datei beschreibt die wichtigsten Infos rund um dein Projekt. Ihr Inhalt wird daher in Github direkt unter der Dateiliste angezeigt. Die Datei nutzt das Markdown-Format.
  • LICENSE: Diese Datei beschreibt die Lizent deines Projekts. Auf opensource.org findest du eine kleine Übersicht mit entsprechenden Downloadlinks. Beachte dabei, dass dein Code auf Github für alle sichtbar ist.
  • Wenn du fertig bist, checke die Änderungen wie anfangs beschrieben in dein lokales Repository ein.
